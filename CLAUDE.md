# CLAUDE.md

Guidance for working in this Claude Code plugin marketplace repository.

## Development commands

```bash
npm run build        # Regenerate .claude-plugin/marketplace.json from plugins/
npm run validate     # Validate all plugins and marketplace manifest
npm test             # Run all tests (node --test tests/*.test.js)
npm run lint         # ESLint + markdownlint
npm run lint:fix     # Auto-fix lint issues
npm run new-plugin   # Scaffold a new plugin
```

Run a single test file: `node --test tests/plugin-structure.test.js`

## Critical: marketplace.json is generated

**Never edit `.claude-plugin/marketplace.json` by hand.** It is auto-generated by `npm run build` (`scripts/build-marketplace.js`), which scans `plugins/` and builds the manifest from each plugin's `plugin.json`.

CI checks for drift: it runs `npm run build` then `git diff --exit-code .claude-plugin/marketplace.json`. If you add or modify a plugin, run `npm run build` and commit the updated manifest.

## Code style

- ESM modules (`"type": "module"` in package.json), use `import`/`export`
- Semicolons required
- Single quotes (with `avoidEscape`)
- Prefix unused function args with `_` (e.g., `_req`, `_index`)
- `no-console` is off — console output is expected in scripts

## CI pipeline

Two workflows run on PRs to `main`:

- **Test** (`.github/workflows/test.yml`): builds marketplace.json, checks for drift, runs `npm run validate`, runs `npm test` — on Node 20.x and 22.x
- **Lint** (`.github/workflows/lint.yml`): ESLint, markdownlint, JSON syntax validation

## Architecture

- **`shared/validation/plugin-validator.js`** — shared validation module used by both `scripts/validate-plugins.js` and the test suite. Exports `validatePlugin()`, `validatePluginEntry()`, `validateMarketplace()`, `validateAllPlugins()`.
- **`scripts/build-marketplace.js`** — discovers plugins by scanning `plugins/*/`, reads each `plugin.json`, detects component directories, and writes `marketplace.json`.
- **Component discovery conventions**: agents require `AGENT.md` in subdirs of `agents/`, skills require `SKILL.md` in subdirs of `skills/`, hooks require `hooks.json` in `hooks/`, commands are `*.md` files in `commands/`.
- **Plugin entries** in marketplace.json must be objects with `name`, `source`, and `description` — string paths are not valid.
