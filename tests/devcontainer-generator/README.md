# Devcontainer Generator Tests

Integration tests designed to run **inside** a devcontainer generated by the devcontainer-generator skill.

## Usage

```bash
# Run all tests
bash tests/devcontainer-generator/run-all.sh

# Run a single test
bash tests/devcontainer-generator/test-firewall-chain-exists.sh
```

## Test Categories

### Firewall Tests
| Test | Description |
|------|-------------|
| `test-firewall-chain-exists.sh` | DEVCONTAINER_FW iptables chain exists |
| `test-firewall-root-exempt.sh` | Root user bypasses firewall |
| `test-firewall-allow-whitelisted.sh` | Whitelisted domains are accessible |
| `test-firewall-deny-blocked.sh` | Non-whitelisted domains are blocked |
| `test-firewall-dns-allowed.sh` | DNS resolution always works |
| `test-firewall-loopback-allowed.sh` | Localhost always works |
| `test-firewall-idempotent.sh` | Double apply produces no duplicates |
| `test-firewall-allow-star.sh` | ALLOW * policy opens all traffic |
| `test-firewall-cidr-rule.sh` | CIDR rules are applied |
| `test-firewall-rules-syntax.sh` | Rules file has valid syntax |

### Structure Tests
| Test | Description |
|------|-------------|
| `test-structure-files-exist.sh` | All required files present |
| `test-structure-scripts-executable.sh` | Scripts have execute permission |
| `test-structure-json-valid.sh` | devcontainer.json is valid JSON |
| `test-structure-cap-net-admin.sh` | NET_ADMIN capability configured |

### Claude Code Tests
| Test | Description |
|------|-------------|
| `test-claude-installed.sh` | Claude Code is installed |
| `test-claude-ccyolo-alias.sh` | ccyolo alias configured |

## Output Format

Each test outputs one line:
- `PASS: <description>` (exit 0)
- `FAIL: <description> - <reason>` (exit 1)
- `SKIP: <description>` (exit 2)
