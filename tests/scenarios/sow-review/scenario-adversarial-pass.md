# Scenario: Adversarial Analysis

## Setup
Provide a SOW with exploitable gaps:
- "Vendor shall provide support as needed" (unbounded obligation)
- Payment tied to calendar dates, not acceptance
- No substitution policy for named team members
- Penalties for vendor delays but none for client delays

## Invocation
"Do an adversarial review of this SOW"

## Expected Behavior
1. Standard dimensional analysis first
2. Client perspective: identifies vendor exploitation vectors
3. Vendor perspective: identifies client exploitation vectors
4. Each gap has a specific fix recommendation

## Acceptance Criteria
- [ ] Client perspective identifies: unbounded support, no substitution controls
- [ ] Vendor perspective identifies: no client delay penalties, payment without acceptance
- [ ] Each adversarial challenge has: scenario description, exploitation risk, specific fix
- [ ] Fixes are actionable (specific clause language, not just "add a clause")
